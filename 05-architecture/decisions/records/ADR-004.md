# ADR-004: Auditoría append-only

**Estado:** ACCEPTED
**Fecha:** 2026-06-24
**Autores:** Equipo ADSO 3145555
**Equipos involucrados:** Arquitectura, Seguridad

---

## Contexto

El sistema Horarios SENA necesita registrar todas las acciones críticas para cumplir con requisitos de auditoría y trazabilidad:

- Quién realizó la acción.
- Cuándo la realizó.
- Qué cambió (valor anterior y nuevo).

La auditoría debe ser inmutable: una vez registrada, no se puede modificar ni eliminar.

## Decisión

Se implementa una tabla audit_log de tipo append-only en PostgreSQL.

### Detalles

- Tabla: audit_log
- Campos: id, user_id, action, entity_type, entity_id, old_value (JSON), new_value (JSON), ip_address, timestamp.
- Inserción: Solo se permiten INSERT. No hay UPDATE ni DELETE.
- Propagación: Se usa Spring AOP para interceptar métodos anotados con @Auditable.
- Servicio: El audit-service recibe los eventos de auditoría y los persiste.

### Estructura de la tabla

- id: UUID, clave primaria
- user_id: UUID, usuario que realizó la acción
- action: VARCHAR(20), CREATE, UPDATE, DELETE, LOGIN, LOGOUT
- entity_type: VARCHAR(50), entidad afectada
- entity_id: UUID, ID del registro afectado
- old_value: JSONB, valor anterior
- new_value: JSONB, valor nuevo
- ip_address: VARCHAR(45), IP desde donde se realizó la acción
- timestamp: TIMESTAMP con zona horaria

### Eventos que se auditan

- CREATE: Instructor, Aprendiz, Ambiente, Horario, Programa, etc.
- UPDATE: Cualquier entidad modificada.
- DELETE: Cualquier entidad eliminada (soft-delete).
- LOGIN y LOGOUT: Usuario.

## Consecuencias

### Positivas

- Trazabilidad completa de todas las acciones críticas.
- Cumplimiento normativo (Ley 1581 de protección de datos).
- Imposibilidad de eliminar evidencia de auditoría.

### Negativas / Trade-offs

- Mayor consumo de espacio en disco (los logs crecen con el tiempo).
- Pequeña penalización de rendimiento por la escritura de logs.

### Riesgos

- El espacio en disco debe ser monitoreado y planificar purga/archivo de logs antiguos.

## Alternativas consideradas

| Alternativa | Por qué se descartó |
|-------------|---------------------|
| Usar logs de aplicación | No es estructurado ni fácil de consultar. |
| Event sourcing | Demasiado complejo para el alcance actual. |
| Tabla de auditoría mutable | No garantiza inmutabilidad. |
