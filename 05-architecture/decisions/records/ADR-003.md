# ADR-003: Validación de conflictos en tiempo real

**Estado:** ACCEPTED
**Fecha:** 2026-06-24
**Autores:** Equipo ADSO 3145555
**Equipos involucrados:** Arquitectura, Dominio

---

## Contexto

El sistema Horarios SENA debe garantizar que no existan conflictos de horario:

- Un instructor no puede estar asignado a dos horarios que se solapen.
- Un ambiente no puede estar asignado a dos horarios que se solapen.
- Una ficha no puede tener dos horarios que se solapen.

La validación debe ser inmediata (en tiempo real) para evitar que el coordinador guarde un horario conflictivo.

## Decisión

La validación de conflictos se realiza en tiempo real en la capa de servicio (Application), usando consultas a la base de datos.

### Detalles

- Lugar: En el ScheduleService antes de persistir el horario.
- Método: Se consulta la base de datos para verificar si existen horarios solapados para el mismo instructor, ambiente o ficha.
- Consulta: Se usa JPA con criterios de búsqueda (fecha, hora inicio, hora fin).
- Respuesta: Si hay conflicto, se lanza una excepción ConflictException que es manejada por GlobalExceptionHandler.

## Consecuencias

### Positivas

- Validación en tiempo real: el usuario recibe feedback inmediato.
- Consistencia de datos: no se persisten horarios conflictivos.
- Centralizado: la validación está en un solo lugar.

### Negativas / Trade-offs

- Mayor carga en la base de datos por las consultas de validación.
- Posible cuello de botella en alta concurrencia (optimizar con índices).

### Riesgos

- Las consultas de validación deben ser eficientes (usar índices compuestos).
- En condiciones de alta concurrencia, podría haber condiciones de carrera (usar @Transactional con ISOLATION.SERIALIZABLE o bloqueo optimista).

## Alternativas consideradas

| Alternativa | Por qué se descartó |
|-------------|---------------------|
| Validación en base de datos (constraints) | PostgreSQL no tiene soporte nativo para constraints de exclusión con tiempos. |
| Validación asíncrona | El usuario no recibiría feedback inmediato. |
| Validación en frontend | No es confiable porque los datos pueden cambiar. |
