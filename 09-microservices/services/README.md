# Catálogo de servicios (documentación)

Este directorio contiene documentación por servicio. Para añadir un nuevo microservicio, crear una carpeta con el nombre del servicio o un archivo `<service>.md` siguiendo la plantilla en `../_template/`.

Estructura recomendada por servicio:

- `api-contract.md` — Contrato HTTP / endpoints principales.
- `data-model.md` — Modelo de datos y esquemas relevantes.
- `events.md` — Eventos publicados y consumidos por el servicio.
- `runbook.md` — Procedimientos de operación y recuperación.

Plantilla de servicio: [09-microservices/_template/README.md](../_template/README.md)
